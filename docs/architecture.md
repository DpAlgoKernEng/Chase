## 概述

Chase 是一个高性能、生产级的 HTTP/1.1 服务器库，采用 **SO_REUSEPORT + 多进程架构**，实现零 IPC 开销的连接分发。项目使用 C 语言实现核心，并提供 C++ API 封装层。

**核心特性**：

- 跨平台事件驱动架构（epoll/kqueue/poll）
- Master/Worker 多进程架构，内核级负载均衡
- 完整 HTTP/1.1 支持（Keep-Alive、Chunked、Range）
- SSL/TLS 支持（OpenSSL 1.1.1 + 3.x 兼容）
- 虚拟主机与 SNI 支持
- DDoS 防护与安全审计日志
- 配置热更新支持

### SO_REUSEPORT 原理说明

SO_REUSEPORT 是 Chase 多进程架构的核心技术，实现零 IPC 开销的连接分发。

#### 传统架构（高 IPC 开销）

```
+============================================================================+
|                     传统架构: Master accept + IPC 分发                     |
+============================================================================+

    +------------------+
    |   Master 进程    |
    +------------------+
           |
           | accept() 获取连接
           v
    +------------------+
    |   client_fd      |
    +------------------+
           |
           | 选择目标 Worker
           v
    +------------------+
    | sendmsg()        |
    | SCM_RIGHTS       |
    |                  |   ~100μs IPC 开销
    | 通过 Unix Socket |   性能瓶颈！
    | 传递 fd          |
    +------------------+
           |
           v
    +------------------+      +------------------+      +------------------+
    |   Worker 1       |      |   Worker 2       |      |   Worker N       |
    +------------------+      +------------------+      +------------------+
```

#### SO_REUSEPORT 架构（零 IPC 开销）

```
+============================================================================+
|                     SO_REUSEPORT: 内核级负载均衡                           |
+============================================================================+

    +------------------+      +------------------+      +------------------+
    |   Worker 1       |      |   Worker 2       |      |   Worker N       |
    +------------------+      +------------------+      +------------------+
           |                         |                         |
           | socket()                | socket()                | socket()
           | + SO_REUSEPORT          | + SO_REUSEPORT          | + SO_REUSEPORT
           | bind(:8080)             | bind(:8080)             | bind(:8080)
           | listen()                | listen()                | listen()
           |                         |                         |
           v                         v                         v
    +------------------+      +------------------+      +------------------+
    |   accept()       |      |   accept()       |      |   accept()       |
    |                  |      |                  |      |                  |
    |   <-- 内核分发   |      |   <-- 内核分发   |      |   <-- 内核分发   |
    +------------------+      +------------------+      +------------------+
           ^                         ^                         ^
           |                         |                         |
           +-------------------------+-------------------------+
                                     |
                                     v
                            +------------------+
                            |   内核负载均衡   |
                            |                  |
                            | - Round-Robin    |
                            | - 连接分发       |
                            | - 零 IPC 开销    |
                            +------------------+

    关键优势:
    - 无 IPC: Worker 直接 accept，无需 Master 介入
    - 内核均衡: 内核自动 Round-Robin 分发新连接
    - 进程隔离: 单 Worker 崩溃不影响其他
    - 简单调试: 独立进程，无锁竞争
```

#### SO_REUSEPORT 使用示例

```c
// 创建 SO_REUSEPORT socket
int create_reuseport_socket(int port, const char *bind_addr, int backlog) {
    int fd = socket(AF_INET, SOCK_STREAM, 0);
    
    // 启用 SO_REUSEPORT（关键！）
    int opt = 1;
    setsockopt(fd, SOL_SOCKET, SO_REUSEPORT, &opt, sizeof(opt));
    
    // 也启用 SO_REUSEADDR（地址复用）
    setsockopt(fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
    
    // 绑定地址
    struct sockaddr_in addr;
    addr.sin_family = AF_INET;
    addr.sin_port = htons(port);
    addr.sin_addr.s_addr = bind_addr ? inet_addr(bind_addr) : INADDR_ANY;
    
    bind(fd, (struct sockaddr*)&addr, sizeof(addr));
    listen(fd, backlog);
    
    return fd;
}

// 每个 Worker 调用相同代码，内核自动分发
void worker_run(int worker_id, int port) {
    int server_fd = create_reuseport_socket(port, NULL, SOMAXCONN);
    
    // 创建 EventLoop
    EventLoop *loop = eventloop_create(1024);
    
    // 注册 accept 回调
    eventloop_add(loop, server_fd, EV_READ, on_accept, NULL);
    
    // 运行事件循环
    eventloop_run(loop);
}
```

## 连接状态机

### 状态定义

```c
typedef enum {
    CONN_STATE_CONNECTING,      // 正在连接
    CONN_STATE_SSL_HANDSHAKING, // SSL/TLS 握手进行中

## EventLoop 工作原理

### 跨平台后端选择

Chase EventLoop 支持三种 I/O 多路复用后端，根据平台自动选择最优方案：

```
+============================================================================+
|                           EventLoop 后端选择                               |
+============================================================================+

    +------------------+
    |   EventLoop      |
    |   创建           |
    +------------------+
           |
           v
    +------------------+
    |   平台检测       |
    +------------------+
           |
           +----------------+----------------+
           |                |                |
           v                v                v
    +-------------+  +-------------+  +-------------+
    |   Linux     |  | macOS/BSD   |  |   其他      |
    +-------------+  +-------------+  +-------------+
           |                |                |
           v                v                v
    +-------------+  +-------------+  +-------------+
    |   epoll     |  |   kqueue    |  |   poll      |
    |             |  |             |  |             |
    | - epoll_create1|  | - kqueue() |  | - poll()   |
    | - epoll_ctl   |  | - kevent() |  | - pollfd[] |
    | - epoll_wait  |  |             |  |             |
    +-------------+  +-------------+  +-------------+

    性能对比:
    - epoll:  O(活跃fd)  Linux 最佳
    - kqueue: O(活跃fd)  macOS/BSD 最佳
    - poll:   O(totalfd) 通用但性能较低
```

    +----------------+
    | kqueue()       |
    +----------------+
           |
           v
    +----------------+
    | 分配 entries   |
    | 数组           |
    +----------------+

    添加事件监听:
    +----------------+
    | eventloop_add  |
    +----------------+
           |
           v
    +----------------+
    | kevent()       |
    | EV_SET         |
    |                |
    | struct kevent ev;
    | EV_SET(&ev, fd, 
    |   EVFILT_READ, 
    |   EV_ADD|EV_ENABLE,
    |   0, 0, NULL); |
    | kevent(kq, &ev,|
    |   1, NULL,0,NULL);|
    +----------------+

    事件循环:
    +----------------+
    | eventloop_run  |
    +----------------+
           |
           v
    +----------------+
    | while(running) |
    +----------------+
           |
           v
    +----------------+        +----------------+
    | kevent()       | <----- | timespec      |
    | (等待事件)     |        | timeout       |
    +----------------+        +----------------+
           |
           v
    +----------------+
    | 处理返回的     |
    | kevent 数组    |
    +----------------+
           |
           v
    +----------------+
    | 检查 filter    |
    | - EVFILT_READ  |
    | - EVFILT_WRITE |
    +----------------+
           |
           v
    +----------------+
    | 调用对应       |
    | callback       |
    +----------------+

    kqueue 特点:
    - 支持 EV_DISABLE 禁用而非删除
    - 支持 EV_DELETE 移除 filter
    - EV_EOF 标识连接关闭
    - EV_ERROR 标识错误
```

### 性能瓶颈分析

```
+============================================================================+
|                           性能瓶颈分析                                     |
+============================================================================+

    传统架构瓶颈:
    +------------------+
    | Master accept    |
    | + IPC 分发       |
    +------------------+
           |
           | ~100μs IPC 开销
           v
    +------------------+
    | 性能瓶颈         |
    | (连接分发延迟)   |
    +------------------+

    Chase 解决方案:
    +------------------+
    | SO_REUSEPORT     |
    | 架构             |
    +------------------+
           |
           | 0 IPC 开销
           v
    +------------------+
    | 内核负载均衡     |
    | (Round-Robin)    |
    +------------------+

    其他优化点:
    - EventLoop: O(活跃fd) 复杂度
    - Buffer: 环形缓冲区，避免频繁分配
    - Timer: 最小堆实现，O(log n) 插入/删除
    - Security: 分片哈希表 (16 shards)，降低锁竞争
```
